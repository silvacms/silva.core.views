Changelog
=========

2.3.2-bethel.2 (unreleased)
---------------------------

* added 'other' to the test AbsoluteURLRequest

2.3.2-bethel.1 (2011-08-29)
------------------------------

* merge changeset: 0de42b7de878, bringing code up to version 2.3.2 with
  an additional fix for breadcrumbs in localsites

* fix get_root in site.py -- the zope app root default view was broken 
  (in this code) if there are no silva roots

2.3.2 (2011/07/27)
------------------

* Fix a bug while truncating a title in the breadcrumb.

* Fix URL computation on error pages.

2.3.1-bethel.1 (2011-07-07)
---------------------------

* adjusted absoluteurl/@@breadcrumbs so that it returns the 'id' and 'obj'
  for each breadcrumb.  This data is used in Bethel's smi pathbar/breadcrumb.
  
* adjust the HTTPHeaderView to __call__ the adapter rather than call 
  setupHeaders directly (enabling overriding adapters to put overriding code
  in __call__).

* adjust views.Page.content to wrap non-IContentLayout content around
  a default content template
  
* add a ContentTemplateMixin, which factors out wrapping the rendered content
  in a content layout template
  
* adjust absoluteurl.url() to strip trailing 'index' section from urls
  for index documents (url canonicalization)
  
* adjust absoluteurl.url() to append trailing '/' to IContainers
  (url canonicalization)

2.3.1 (2010/12/20)
------------------

* Update interface documentation.

* Fix a bug where a silvaviews.View would not render the expected content.

2.3 (2010-01-07)
----------------

* Update imports due to API changes.

* Update test setup, use new test infrastructure.

* Properly set HTTP headers on Pages as well.

2.3b1 (2010/07/16)
------------------

* Form code move to ``silva.core.forms``.

* Layout system is now based on ``infrae.layout``.

* HTTP headers have been improved to be customized, HEAD requests should set
  the Content-Length of the content if possible.

* absolute_url is now available on versions and test requets.

* Update to match publisher changes in Zope 2.12.


2.2b1 (2010-01-22)
------------------

* Add some support to set HTTP headers on content and views.

* Manage HEAD requests on content and views.

* Create a layout support by using five.megrok.layout.

* Add support to create public forms.

* Add the possibility to change widget rendering style between
  different z3c forms.

* Improve form and form templates, more support for status messages.

* Add the possiblity to remove the cancel button on z3c forms by using
  an interface.

* SMI forms should be registered on the SMI layer.

* Use silva.translations for translations support.


2.2a2 (2009-01-12)
------------------

* Add generic ZMI form.

* Update forms macros.

* Use five.grok viewlets instead of a local implementation.


2.2a1 (2008-11-18)
------------------

* Initial release
